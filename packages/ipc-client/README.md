# @loam-app/ipc-client

The typed IPC surface between the React frontend and the Tauri shell (§5.4).

## Layout

- `src/generated/bindings.ts` — **generated by tauri-specta from the Rust
  command surface. Never edit manually.** Commands return
  `{ status: "ok", data } | { status: "error", error: LoamError }`.
- `src/events.ts` — typed subscription helpers for the §5.4 event channels
  (`vault://file-changed`, `vault://index-progress`, `vault://conflict`),
  payloads wrapped in `EventEnvelope { seq, vaultId, payload }`.
- `src/transport.ts` — the transport seam: the frontend never touches Tauri
  globals directly and always runs in a plain browser (mock transport; the
  full in-memory mock is LOA-64).
- `src/contract-checks.ts` — compile-time drift tripwires (`@ts-expect-error`
  on stale call shapes; `pnpm typecheck` fails if the contract moves).

## Regenerating the bindings

The bindings are committed and drift-checked in CI (the
`bindings_drift` test fails on any divergence). After changing the Rust
command surface:

```sh
LOAM_UPDATE_FIXTURES=1 cargo test -p loam-desktop --test bindings_drift
```

then review and commit the diff. Contract changes follow the process in
`docs/ipc-contract.md` (LOA-65): additions are compatible; renames and
removals are breaking and need a documented migration.
